ServerPush by markathomas

ServerPush add-on for Vaadin

Dependencies

Atmosphere Framework 0.7.2+

Install

  1. Add JARs in add-on archive to classpath (should also go in WEB-INF/lib)
  2. Create new widgetset if you do not already have a custom widgetset and inherit the org.vaadin.addons.serverpush.ServerPushWidgetset widgetset
  3. Compile new widgetset as part of your build and add to the VAADIN/widgetsets/ directory in the root of your web application
  4. Add the contents of the "web" directory from the add-on archive; merging if you need to. Your web.xml should have both AtmosphereServlet and the add-on's Vaadin servlet configured. atmosphere.xml should be in META-INF.
  5. Change servlet mapping for Vaadin servlet if you so desire, but leave the servlet mapping for AtmosphereServlet alone.
  6. Modify Vaadin servlet init-params to use your Application class and your custom widgetset
  7. Add the ServerPush widget to your Application if not using the ServerPushApplicationServlet
  8. Perform some action on the server that changes state on the client (e.g. add a new item to a table's container)
